# 04. サイト設計 — ページツリーとデザイン方針(要望3)

最終更新: 2026-07-10 / 状態: **確定**(掲載指標は docs/03-metrics-ranking.md の選定結果を反映済み)

## 1. 設計コンセプト

ユーザー要望「1ページで本日分の一覧 + 過去分はテーマごとの個別ページ」を採用しつつ、
差別化の核である「寄与度」を全ページの共通フォーマットに組み込む。

**共通フォーマット(全指標カード共通)**: `現在値` + `歴史的位置(パーセンタイル)` + `寄与度エビデンス(効果目安と出典)` の3点セット。
これにより「数値は見れるが意味が分からない」を構造的に解消する。

3点セットの適用粒度(2026-07-10 T-12レビューを受けて明文化):
- **指標を主題として提示する文脈**(ファクター個別ページ、市場体温計)ではフル3点セット必須(個別ページではセクション構成での充足も可: 現在値=該当リスト/リターン、歴史的位置=過去推移、エビデンス=専用節)
- **一覧・導線カード**(トップの天気図カード等)は簡約形とするが、`寄与度エビデンス要約1行+出典リンク` を必ず含める(現在値のみのカードは禁止)。歴史的位置は遷移先の個別ページが担う
- **銘柄リストの行**は指標カードではないため3点セット対象外(リスト全体の冒頭にエビデンス注記があれば足りる)

## 2. ページツリー

```
/ (本日のダッシュボード)
├── /markets/
│   ├── /markets/jp/          … 日本市場サマリ(指数・バリュエーション・信用残の現在地と推移)
│   └── /markets/us/          … 米国市場サマリ(同上: S&P500 PER、金利、Fear&Greed的な位置)
├── /factors/                  … ファクター(テーマ)一覧
│   └── /factors/{factor}/     … ファクター個別ページ。初期リリースは6本:
│       value / momentum / dividend / quality / size (Tier 1)
│       + margin-trading (信用倍率。「需給参考」ラベルの別デザインで隔離)
│       ├── 解説(そもそも何か、なぜ効くとされるか)
│       ├── 寄与度エビデンス(文献+自前簡易検証の結果)
│       ├── 過去推移(そのファクターの調子: 直近1年/5年のファクターリターン)
│       └── 本日の該当銘柄リスト(スクリーニング結果、日米タブ切替)
├── /screener/                 … 本日の全スクリーニング結果一覧(ファクター横断)
├── /learn/                    … 指標の読み方・用語集(SEO兼ペルソナ教育コンテンツ)
│   └── /learn/{term}/
├── /about/                    … サイトの目的・検証方法の透明性ページ(信頼の担保)
└── /disclaimer/               … 免責(全ページフッターからリンク。投資助言ではない旨)
```

### トップページ(本日のダッシュボード)の構成
1. ヘッダー: 日付、日米市場のひとこと状況(数値ベース、扇動的表現禁止)
2. **市場体温計**: 日米それぞれの市場全体バリュエーションの歴史的パーセンタイル(帯グラフ)
3. **ファクター天気図**: 各ファクターの直近リターン(今どのファクターが効いているか)を一覧カード表示。カード→個別ページへ
4. **本日の注目シグナル**: スクリーニング該当件数の変化が大きいもの(例:「PBR最低位クォンタイル入り銘柄が前日比+12件」)
5. フッター: 免責、データ出典、最終更新時刻

### 個別ページの「過去分の集積」
要望どおりテーマ(ファクター)単位で集積する。日次スナップショットはビルド時にJSON/CSVで蓄積し、
個別ページで時系列チャート化する(kb-markdown-datastore.md のパターンを流用)。

## 3. 採用しなかった代替案と理由

- **銘柄個別ページ(全銘柄分)**: 既存サイトと正面競合しコモディティ。ファクター軸の集積のほうが差別化になる。P4以降で「該当銘柄→ミニページ」を検討
- **リアルタイム更新**: 日次で十分(ペルソナはデイトレーダーではない)。静的サイトのコスト優位を優先

## 4. デザイン方針

- トーン: 「金融ダッシュボード」寄りの静かで信頼感のあるデザイン。扇動色(急騰!等の赤字強調)を使わない
- カラー: ダーク/ライト両対応。上昇/下落の色は日米ユーザー混在を考慮し **上昇=緑/下落=赤(国際標準)** に統一し、初回訪問時に注記
- タイポグラフィ: 数値が主役。等幅数字(tabular-nums)を使用
- チャート: 静的生成のSVG(ビルド時生成)を基本とし、JSは最小限。dataviz スキルの規約に従う
- レスポンシブ: モバイルファースト(個人投資家の閲覧は朝の通勤時間帯を想定)
- 全ページ共通: フッターに免責+データ出典+「本サイトは情報提供であり投資助言ではありません」

## 5. 技術スタック(P1以降の実装前提)

- 静的サイトジェネレータ: **Astro**(コンテンツ主体・JS最小・Markdown/JSONデータ駆動に強い)
- データパイプライン: Python(yfinance / J-Quants API無料枠)→ JSON出力 → Astroビルド
- ホスティング/CI: GitHub Pages + GitHub Actions(日次cron)。無料枠内
- データ蓄積: リポジトリ内 data/ にJSON追記(kb-markdown-datastore.md パターン)
