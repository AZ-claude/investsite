## ②3点セット表示(現在値+歴史的位置+寄与度エビデンス) — **FAIL(中)**

代表ページ: `dist/index.html`(トップ)、`dist/factors/value/index.html`(ファクター個別)を実HTMLで確認。

**factorsページ**: 「解説」「寄与度エビデンス」(出典リンク付き)「過去推移」(歴史的位置に相当する
分位ポートフォリオのリターン)「本日の該当銘柄リスト」の4セクション構成で、3要素は**ページ内には
揃っている**。ただし「本日の該当銘柄リスト」の各行は「順位/銘柄/指標値」のみで、
銘柄ごとの歴史的位置(パーセンタイル)や寄与度エビデンスは行に付随していない(ページ上部の
共通エビデンス文言を参照する構造)。

**トップページ**: 「ファクター天気図」セクション(`FactorWeatherCard`)を実HTMLで確認したところ、
表示要素は「ファクター名 / 直近1ヶ月リターン(現在値相当)/ 該当銘柄数」のみで、
**歴史的位置(パーセンタイル)も寄与度エビデンス(出典)もカードに存在しない**(リンク先の個別ページで
初めて出典が出る)。

さらにコード調査で、`site/src/components/MetricCard.astro` というコンポーネントが
存在し、そのdocstringには明記で「現在値 + 歴史的位置(パーセンタイル帯) + 寄与度エビデンス
要約(出典リンク付き)の3点セット」を実装すると書かれている(T-07の完了条件に対応するもの)。
しかし実際に本番ページで使われているか grep で確認したところ:

```
$ grep -rl "MetricCard" site/src/pages/
site/src/pages/dev/catalog.astro
site/src/pages/factors/[slug].astro   # ※コメント内の言及のみ、実際のimport/使用なし

$ grep -c "metric-card" site/dist/index.html
0
$ grep -c "metric-card" site/dist/factors/value/index.html
0
$ grep -c "metric-card" site/dist/dev/catalog/index.html

```

`MetricCard` は `dev/catalog` にしか実際にレンダリングされておらず、本番の `index.astro` /
`factors/[slug].astro` はいずれも独自レイアウト(PercentileBar・独自セクション)で代替実装している。
`factors/[slug].astro` 内のコメントは「MetricCard reference variant と同様の方針」とだけ書いており
実装として呼び出してはいない。

**判定根拠**: docs/04-site-design.md 1節は「共通フォーマット(全指標カード共通)」と明記しており、
これは全ページ共通の構造契約のはず。しかし本番導線の代表例(トップの天気図カード)ではこの3点が
1カードに揃っていない。情報の欠落(出典が消えている)ではなく構造の欠落(カード単位で3点セットに
なっていない)なので実害は限定的だが、ドキュメントの完了条件("全共通コンポーネントがダミーデータで
表示される")は満たしていても、本番ページへの統合は未完了という状態。
