# T-01 スパイクレポート: 日本株の無料データ取得

実行日: 2026-07-10(JST) / 実行者: 実作業員(Sonnet) / 再現スクリプト: `pipeline/spikes/t01_jp_data.py`

## 結論(先に要点)

**日本株の「本日の」Tier1指標5種は、yfinance(東証ティッカー `XXXX.T`)を採用すれば無料で成立する。J-Quants無料プランは12週遅延のため「本日の」指標には使えない(不採用)。**

| 指標 | 採用ソース | 遅延 | 取得できた銘柄割合(実測) |
|---|---|---|---|
| 株価 | yfinance (`.info`/`fast_info`) | 前営業日終値(当日ザラ場は取れない。日本市場終値の反映は当日夜〜翌朝JST) | 226/226 = 100%(日経225全銘柄) |
| PER(trailingPE) | yfinance `.info['trailingPE']` | 同上 | 212/226 = 93.8%(欠損14銘柄、後述) |
| PBR(priceToBook) | yfinance `.info['priceToBook']` | 同上 | 226/226 = 100% |
| 時価総額 | yfinance `.info['marketCap']` / `fast_info.market_cap` | 同上 | 226/226 = 100%(値は取れるが**精度に留保あり**、後述) |
| 12-1ヶ月モメンタム用の株価履歴 | yfinance `.history(period="14mo")` | 同上 | 10/10(サンプル検証銘柄で全件287行、2025-05-09〜2026-07-09の日次データ取得成功) |

**ただし2点の重要な留保事項がある(下記「留保事項」参照)**:
1. 時価総額はYahoo!ファイナンス公式表示値と最大約19%乖離するケースを確認(トヨタ)。発行済株式数の反映タイミング/自己株式控除の扱いの差と推定(未確定)。
2. yfinanceの`trailingPE`はYahoo!ファイナンスJPの「PER(会社予想)」とは定義が異なる(実績PERベース vs 会社予想EPSベース)。サイトに掲載する際は「PERの定義: 直近実績ベース」等の注記が必要。

## 1. J-Quants API無料プラン(ドキュメント確認、実測未)

登録・APIキー発行が必要なため、本タスクではAPI呼び出しの実測は行っていない。以下は公式サイト・公式ニュースリリースの一次情報。

- 出典1: https://jpx-jquants.com/ja (J-Quants公式サイト、料金プランページ、2026-07-10時点確認)
- 出典2: https://www.jpx.co.jp/corporate/news/news-releases/6020/20250505-01.html (JPX公式ニュースリリース)

**Freeplanの内容(公式表記そのまま)**:
- 取得可能過去データ: **直近12週間を除く2年間**
- APIコール制限: 5件/分
- CSVダウンロード: 不可
- データ種別: 上場銘柄一覧 / 株価四本値(日通し) / 財務情報(サマリーのみ) / 決算発表予定日 / 取引カレンダー
- TOPIX四本値・指数四本値・信用取引データ等はFreeプラン対象外(Light以上、月額1,650円〜)

**公式FAQ抜粋**:
> Q. 無料プランでは当日の株価を取得できないのでしょうか？
> A. できません。無料プランではデータは12週間遅延して配信されます。

**判定**: 03-metrics-ranking.md 4節で懸念されていた「12週遅延の疑い」は**公式ドキュメントで確定**。「本日の」指標という完了条件に対してJ-Quants Freeプランは構造的に不成立。財務情報がサマリーのみでPER/PBRの元となるEPS/BPSが直接提供されるかは実測できず未確認だが、株価自体が12週遅延する時点でこの用途には使えない。

## 3. 日経225全銘柄の一括取得(所要時間・失敗率・レート制限)

### 3-1. 日経225構成銘柄リストの無料取得元

**採用**: 日経平均プロフィル公式サイトのウェイトCSV(登録不要、無料)
`https://indexes.nikkei.co.jp/nkave/archives/file/nikkei_stock_average_weight_en.csv`

- エンコーディングはUTF-8ではなくCP932(Shift-JIS系)。デコード時に注意が必要(スクリプトでフォールバック処理済み)。
- 取得行数は227行(ヘッダ1行+銘柄225行+著作権表示のフッタ1行)。フッタ行を除けば**ちょうど225銘柄**、想定通り。

## 4. 実施中に発生した環境問題と対処(2回以内で自力解決)

1. **1回目の躓き**: インストール済みのyfinance 0.2.51で全リクエストが`429 Too Many Requests`エラー・JSONDecodeErrorとなり、最初の1銘柄すら取得できなかった。原因はYahoo Finance側のボット対策強化に旧バージョンのyfinanceが対応できていないため(推定、公式リリースノートで直接確認はしていない)。
2. **対処**: `pip install --upgrade yfinance` で **0.2.51 → 1.5.1** にアップグレード(内部でブラウザ偽装ライブラリ`curl_cffi`を追加導入)。アップグレード後は10銘柄・225銘柄とも安定して取得できた。
3. **CSVエンコーディングエラー**: 日経公式CSVをUTF-8としてデコードしようとして失敗。CP932へのフォールバックで解決(3-1節参照)。

**運用上の含意**: T-05以降の日次パイプライン実装では、**yfinanceを最新版に固定(ピン留め)し、定期的な動作確認を組み込む**必要がある(Yahoo Finance側の非公式API仕様変更に追随する運用コストが発生する)。

## 5. 留保事項(未解決のまま引き継ぐ点)

1. **時価総額の精度**: トヨタで19%の乖離を観測。SBG・ソニーでは1〜2%差で実用上問題ないレベルだったが、トヨタのケースは無視できない差。仮説(未検証): yfinanceの発行済株式数データが古い/自己株式控除の有無が異なる。**T-04(指標計算モジュール実装)着手前に、複数銘柄でのサンプリング検証を追加することを推奨**(本タスクの非スコープのため未実施)。
2. **PERの定義差**: yfinanceの`trailingPE`は実績ベース、Yahoo!ファイナンスJPの主要表示は会社予想ベース。サイトでどちらを採用するか、またその定義をどうユーザーに開示するかはT-03(データスキーマ確定)で決定する必要がある。
3. **J-Quants財務情報(サマリー)の実際の中身**: 未登録のため未確認。将来的にPBR改革の開示状況等、財務情報が必要になった場合はLight以上のプラン検討、または登録して実測することを推奨。
4. **TOPIX500相当の一括取得は日経225(225銘柄)で代替検証**。完全な500銘柄でのレート制限挙動は未実測(TOPIX500構成銘柄リストの無料入手性自体は未調査)。約2倍の銘柄数でも同様の傾向(レート制限なし)と推定されるが、これは推測であり実測ではない。
